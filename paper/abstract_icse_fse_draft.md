# VulnSynth Abstract Draft (ICSE/FSE Style)

## Version 1

Large language models (LLMs) have recently shown promise in synthesizing static analysis rules from vulnerability reports and patches, but generating reliable CodeQL detectors remains challenging. Existing approaches largely treat CVE-to-CodeQL synthesis as a black-box end-to-end generation problem, where failures are handled by repeatedly rewriting entire queries from compiler and execution feedback. This trial-and-error process often suffers from unstable convergence, weak failure attribution, and high repair cost, especially on long-tail vulnerabilities that do not fit a single taint-style reasoning pattern. We present **VulnSynth**, a framework that reformulates CVE-to-CodeQL detector synthesis as **structured program synthesis over layered semantic states**. VulnSynth first derives a three-level intermediate representation: `L1` grounds vulnerability-relevant facts and patch evidence, `L2` captures vulnerability semantics through typed reasoning templates, and `L3` specifies a dependency-aware query construction plan. It then performs **step-wise compositional lowering**, generating CodeQL fragments for individual `L3` steps before composing them into a final executable query. To improve robustness, VulnSynth closes the loop with **execution-guided IR-level repair**: compilation, execution, and evaluation signals are interpreted as structured patches over intermediate states, which are applied under a lightweight `patch_policy` to support selective regeneration and conservative fallback. This design turns query synthesis from black-box query rewriting into a controllable, repairable, and reproducible process. We implement VulnSynth for Java CodeQL and evaluate it on a benchmark of real-world CVE patches. Results show that layered semantic states and IR-level repair improve detector synthesis stability, reduce ineffective regeneration, and provide better long-tail robustness than end-to-end query generation baselines.

## Version 2

Automatically synthesizing CodeQL detectors from CVE patches is difficult because the task entangles patch understanding, vulnerability modeling, query construction, and iterative repair in a single black-box generation loop. As a result, existing LLM-based approaches often rely on repeatedly rewriting full queries from validation feedback, which makes failures hard to localize and repairs expensive. We present **VulnSynth**, which recasts CVE-to-CodeQL synthesis as a **layered, compositional, and repairable** synthesis problem. VulnSynth introduces a three-level semantic state space: `L1` extracts grounded vulnerability facts from patch evidence, `L2` models the vulnerability with typed semantic templates and constraints, and `L3` lowers these semantics into a step graph that defines how a detector should be assembled. Based on this state space, VulnSynth generates CodeQL in two stages: step-level fragment synthesis and final query composition. When validation fails, VulnSynth does not simply regenerate the whole query; instead, it interprets feedback as a small structured patch over `L1/L2/L3`, applies the patch under a lightweight policy over `file/op/path`, and selectively regenerates only the affected downstream representations. This enables failure attribution, localized repair, and robust fallback without introducing a complex repair DSL. We implement VulnSynth on top of CodeQL for Java and evaluate it on real CVE patches. Our results indicate that explicit layered states and policy-constrained repair yield a more stable convergence process and improve synthesis effectiveness on difficult cases compared with end-to-end query-level refinement.

## Version 3 (Vulnerability Semantics Compiler Narrative)

Synthesizing accurate CodeQL detectors from CVE patches remains challenging because the task requires jointly inferring vulnerability semantics, selecting an appropriate reasoning pattern, and constructing executable queries under frequent compilation and runtime failures. We present **VulnSynth**, a **vulnerability semantics compiler** that compiles patch-derived vulnerability semantics into CodeQL detectors through a multi-stage compilation pipeline. The VulnSynth frontend extracts grounded evidence and anchors from patches into `L1` facts; the middle-end maps these facts into a reusable catalog of typed semantic patterns and constraints in `L2`; and the backend lowers `L2` into a dependency-aware construction plan (`L3`) that is compiled into composable CodeQL fragments and finally assembled into an executable query. Unlike end-to-end query rewriting, VulnSynth treats compiler diagnostics and execution feedback as signals over intermediate representations: failures are interpreted as small structured patches (`file/op/path/value`) applied under a lightweight policy, triggering selective recompilation of only the affected downstream stages, with conservative soft/full reset fallbacks for long-tail cases. This compiler-inspired formulation makes detector synthesis controllable, repairable, and reproducible, while reducing ineffective trial-and-error. We implement VulnSynth for Java CodeQL and evaluate it on real-world CVE patches, showing improved robustness and convergence stability over baselines that rely on end-to-end query generation and query-level refinement.
